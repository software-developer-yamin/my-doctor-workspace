- (decision) AD-1 Paradigm: multi-app monorepo — three independent deployment units (Next.js frontend, Express backend, Vite admin SPA) sharing one git workspace. Binds: each unit has own package.json, build pipeline, and package manager. Prevents: shared node_modules, cross-app imports. Rule: ESLint enforces no-import across repo boundaries.
- (decision) AD-2 Auth isolation: three separate auth systems coexist, never share tokens or endpoints. Customer=OTP+JWT (cookies), Doctor=JWT (cookies), Admin=Clerk. Binds: each auth path has dedicated /auth, /customers, /doctors, /clerk endpoints. Prevents: mixing tokens or reusing auth middleware across roles. Rule: verifyAccessToken+protect middleware always both applied on protected routes; admin never calls customer/doctor endpoints.
- (decision) AD-3 API contract shape: all backend responses wrapped as {success, message?, data: T, meta?}. Binds: sendResponse utility is sole HTTP responder — never res.json() directly. Prevents: inconsistent envelope shape breaking frontend ApiResponse<T> typing. Rule: every controller uses sendResponse; errors go to next(error) for centralized handler.
- (decision) AD-4 Adapter boundary: raw MongoDB shape (BackendT with _id) must never reach frontend components. Adapters in src/adapters/ map _id→id and normalize shape. Binds: one adapter per domain entity. Prevents: _id strings leaking into component props; backend schema changes breaking UI. Rule: all API response data passes through adapter before entering component tree or Redux state.
- (decision) AD-5 Frontend state layering (Next.js app only): Redux RTK=global UI+auth only (app slice, auth slice); TanStack Query=all server state (staleTime 60000ms default); nuqs=URL-reflected filter/pagination state; local useState=ephemeral UI only. Prevents: two competing owners of same server data; URL state stored in component memory; auth token in localStorage. Rule: no useEffect+useState for server data fetching.
- (decision) AD-6 Admin panel state: Zustand only (not Redux). Auth via Clerk useAuth()/useUser(). Routes file-based under src/routes/ — routeTree.gen.ts auto-generated, never hand-edited. Package manager: pnpm. Prevents: Redux dependency bleed from frontend into admin; manual route tree edits causing drift.
- (decision) AD-7 Backend module structure: each domain has exactly 4 files (*.controller.ts, *.model.ts, *.routes.ts, *.service.ts). ESM modules: all local imports include .js extension. PascalCase file naming matching domain. All new routes registered in src/routes/routes.ts. Prevents: service-layer password hashing (pre-save hook owns it), catch-and-swallow errors, unregistered routes.
- (decision) AD-8 Asset pipeline: file uploads stored at /uploads/ via Multer; backend serves them at http://localhost:6089/uploads/; Next.js rewrites /uploads/:path* to backend. Frontend always uses relative /uploads/ paths. Prevents: user files served from src/; hardcoded backend host in frontend components.
- (decision) AD-9 Feature gate: frontend pages gated by PAGE_FEATURES map in src/config/features.ts; proxy.ts middleware blocks disabled routes. All new primary routes must be registered there. Prevents: unreachable routes deployed silently; routes accessible before backend integration complete.
- (decision) AD-10 Config centralization: API endpoint strings live exclusively in src/config/api.ts API.ENDPOINTS; cookie/storage key strings in src/config/constant.ts CONSTANT.LOCAL_STORAGE_KEYS. Prevents: hardcoded URL paths or cookie keys scattered in components. Rule: no string literals for API paths or auth cookie keys outside config files.
